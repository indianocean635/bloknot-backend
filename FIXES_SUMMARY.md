### **JavaScript предотвращение скролла:**
```javascript
// Сразу при загрузке
window.scrollTo(0, 0);
document.body.scrollTop = 0;
document.documentElement.scrollTop = 0;

// Отключаем восстановление скролла
if ('scrollRestoration' in history) {
  history.scrollRestoration = 'manual';
}

// После загрузки данных
setTimeout(() => {
  window.scrollTo(0, 0);
}, 100);
```

## 🧪 Как тестировать:

### **1. Перезапустите сервер:**
```bash
pm2 restart ecosystem.config.js
```

### **2. Откройте форму записи:**
- Страница должна начинаться сверху (без автоскролла)
- Хедер должен быть белым с минимальными отступами
- Логотип и название должны быть слева

### **3. Проверьте консоль (F12):**
Должны быть сообщения:
```
"Current path: /book-template/your-slug"
"Extracted slug: your-slug"
"Loading booking data for slug: your-slug"
"Fetching business data..."
"Business response status: 200"
"Business data received: {name: '...', logo: '...'}"
"Company name set to: ..."
"Logo loaded successfully in header"
```

### **4. Если пишет "Загрузка...":**
Проверьте консольные ошибки:
- **404 ошибка:** неверный slug
- **500 ошибка:** проблема с сервером
- **Нет логов:** JavaScript ошибка

### **5. Проверьте мобильную версию:**
- F12 → Mobile view
- Отступы должны быть меньше
- Логотип над названием

## 🚨 Если не работает:

### **Страница скроллит автоматически:**
1. Проверьте что `history.scrollRestoration = 'manual'` выполняется
2. Проверьте что нет `focus()` на input элементах
3. Очистите кэш браузера (Ctrl+F5)

### **Вс еще "Загрузка...":**
1. Проверьте console.log - есть ли ошибки
2. Проверьте API `/api/public/business?slug=...`
3. Проверьте что slug правильный

### **Логотип/название не отображаются:**
1. Проверьте что в настройках они сохранены
2. Проверьте console.log - есть ли данные
3. Проверьте CSS селекторы
