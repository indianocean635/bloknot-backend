## 🎯 Что изменится в БД:

### **Новая модель Schedule:**
```sql
CREATE TABLE "Schedule" (
  "id" INTEGER NOT NULL,
  "type" TEXT NOT NULL,           -- 'owner' или 'master'
  "email" TEXT NOT NULL,          -- email сотрудника
  "masterId" INTEGER,            -- ID мастера (если мастер)
  "businessId" TEXT NOT NULL,    -- ID бизнеса
  "days" JSON NOT NULL,          -- График работы
  "createdAt" DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
);
```

### **Обновления в существующих моделях:**
- **Business** - добавлена связь `schedules`
- **Master** - добавлена связь `schedules`

## ✅ Что исправлено и добавлено:

### **В настройках:**
1. **Раздел "График работы"** - для владельца и мастеров
2. **Календарь** - выбор рабочих дней месяца
3. **Время работы** - настройка часов для каждого дня
4. **Выходные дни** - отметка нерабочих дней

### **В форме записи:**
1. **Белый фон** - убран фиолетовый градиент
2. **iOS-style выбор времени** - как в будильнике iPhone
3. **Разделенные дата и время** - удобнее для пользователей

### **API эндпоинты:**
- `GET /api/schedules` - получить все графики
- `POST /api/schedules` - создать/обновить график
- `GET /api/public/schedule/:email` - получить график для записи
- `GET /api/user` - получить данные текущего пользователя

## 🧪 Тестирование:

### **Настройки графика:**
1. Откройте настройки → График работы
2. Настройте график для владельца
3. Настройте график для мастера
4. Проверьте сохранение

### **Форма записи:**
1. Откройте страницу онлайн-записи
2. Выберите услугу и мастера
3. Попробуйте новый выбор времени (iOS-style)
4. Проверьте белый фон

## 🎨 iOS-style Time Picker:

- **Два колеса** - часы (0-23) и минуты (0, 15, 30, 45)
- **Закругленный дизайн** - как в iPhone
- **Плавная прокрутка** - snap к центру
- **Визуальные индикаторы** - выделение выбранного времени
