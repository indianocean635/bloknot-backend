# Применение миграции БД

## 🚨 СРОЧНО: Исправлена ошибка создания мастера

**Проблема:** PrismaClientValidationError - Unknown argument role
**Решение:** Удалено поле `role` из создания мастера

## Что нужно сделать:

### 1. **Применить миграцию БД:**
```bash
cd c:\Users\User\Desktop\bloknot-backend
npx prisma db push
```

### 2. **Перезапустить сервер:**
```bash
pm2 restart ecosystem.config.js
# или если pm2 не настроен:
npm start
```

## ✅ Что исправлено:

1. **POST /api/masters** - убрано поле `role` из data
2. **PATCH /api/masters** - убрано поле `role` из data  
3. **Валидация** - проверка name и email
4. **Уникальность email** - ошибка P2002 если дубликат

## 🎯 Правильный код создания мастера:

```javascript
const master = await prisma.master.create({
  data: {
    name: body.name,
    email: body.email.toLowerCase(),
    active: true,
    businessId: user.businessId
  }
});
```

## 📋 После миграции будет работать:

1. ✅ **Создание мастера** - только name, email, active
2. ✅ **Валидация email** - формат и уникальность
3. ✅ **Обновление мастера** - без поля role
4. ✅ **Настройки в интерфейсе** - сохранение мастеров

## 🧪 Тестирование:

1. Откройте настройки → Мастера
2. Нажмите "Добавить мастера"
3. Введите имя и email
4. Нажмите "Сохранить"
5. Проверьте что мастер создался

## ⚠️ Если все еще не работает:

1. **Проверьте консоль браузера** - F12 → Console
2. **Проверьте логи сервера** - `pm2 logs`
3. **Убедитесь что миграция применена** - `npx prisma db push`

**Теперь создание мастера должно работать корректно!**
