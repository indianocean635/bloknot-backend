# Применение миграции БД

## Что нужно сделать:

1. **Открыть командную строку** в папке проекта `c:\Users\User\Desktop\bloknot-backend`

2. **Выполнить команду:**
   ```bash
   npx prisma db push
   ```

3. **Перезапустить сервер:**
   ```bash
   pm2 restart ecosystem.config.js
   ```
   или если pm2 не настроен:
   ```bash
   npm start
   ```

## Что изменится:

- Добавится поле `email` в таблицу `Master` (уникальное)
- Добавится поле `categoryId` в таблицу `Service` (опциональное)
- Обновятся связи между моделями

## После миграции:

1. ✅ Услуги будут создаваться с категориями
2. ✅ Мастеры будут требовать email (уникальный)
3. ✅ Все API будут работать правильно
4. ✅ Настройки будут сохранять услуги и мастеров

## Если возникнут ошибки:

1. Проверьте что вы в правильной папке проекта
2. Убедитесь что `DATABASE_URL` настроен в `.env`
3. Проверьте что Prisma клиент установлен: `npm install`
