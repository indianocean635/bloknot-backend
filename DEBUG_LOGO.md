## Как протестировать:

### 1. **Перезапустите сервер:**
```bash
pm2 restart ecosystem.config.js
```

### 2. **Проверьте настройки:**
1. Откройте настройки → Брендинг
2. Введите название компании
3. Нажмите "Сохранить"
4. Проверьте в консоли что нет ошибок

### 3. **Проверьте форму записи:**
1. Откройте страницу онлайн-записи
2. Откройте консоль (F12)
3. Проверьте логи:
   - "Business data:" - должен показать объект с name и logo
   - "Company name set to:" - должно показать название
   - "Logo loaded successfully" - если логотип есть

### 4. **Проверьте результат:**
- ✅ В хедере не должно быть логотипа Bloknot
- ✅ В заголовке должно быть название компании
- ✅ В футере должен быть логотип компании

## Если не работает:

### **Проверьте консоль в браузере:**
1. F12 → Console
2. Ищите ошибки красным цветом
3. Проверьте что business.data содержит name и logo

### **Проверьте API:**
```bash
# Проверить бизнес данные
curl "http://localhost:3000/api/public/business?slug=your-slug"

# Проверить название компании
curl -H "Cookie: session=..." http://localhost:3000/api/business/name
```

### **Проверьте базу данных:**
```bash
npx prisma studio
# Посмотрите таблицы Business и WorkPhoto
```

## Отладочные логи:

В консоли браузера должны быть сообщения:
- "Business response status: 200"
- "Business data: {name: '...', logo: '...'}"
- "Company name set to: ..."
- "Logo loaded successfully" или "No logo found"

Если этих сообщений нет - проблема в загрузке данных.

**После перезапуска сервера всё должно работать!**
