# 🔧 Инструкция по починке сервера

## 📋 **Текущая ситуация:**
- Сервер не запущен
- Сайт и ЛК не открываются
- Нужно диагностировать и исправить проблемы

## 🚀 **Шаг 1: Запустите тест диагностики**

Откройте командную строку в папке `bloknot-backend` и выполните:

```bash
node test-server.js
```

**Что покажет тест:**
- ✅ Загрузка всех модулей
- ✅ Импорты роутов и middleware
- ✅ Создание Express приложения
- ❌ Конкретные ошибки если они есть

## 🚀 **Шаг 2: Если тест прошел успешно**

Запустите основной сервер:

```bash
node index.js
```

Или для разработки:

```bash
npm run dev
```

## 🚀 **Шаг 3: Проверьте что сервер работает**

```bash
# Проверьте health endpoint
curl http://localhost:3001/health

# Или откройте в браузере
http://localhost:3001/health
```

## 🚀 **Шаг 4: Если есть ошибки**

**Пришлите скриншот с логами** из командной строки после выполнения `node test-server.js`

**Возможные проблемы и решения:**

### **Ошибка с Prisma:**
```bash
# Перегенерируйте Prisma клиент
npm run prisma:generate
```

### **Ошибка с переменными окружения:**
```bash
# Проверьте .env файл
cat .env
```

### **Ошибка с портами:**
```bash
# Проверьте что порт 3001 свободен
netstat -an | findstr :3001
```

### **Ошибка с зависимостями:**
```bash
# Переустановите зависимости
npm install
```

## 🚀 **Шаг 5: После запуска сервера**

Проверьте:
1. **API работает:** `http://localhost:3001/health`
2. **Сайт открывается:** Ваш фронтенд должен подключиться к API
3. **ЛК работает:** Панель администратора должна загружаться
