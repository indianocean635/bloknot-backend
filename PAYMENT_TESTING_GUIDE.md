# 🧪 Гид по тестированию платежной системы

## ✅ Проблемы исправлены

### 1. **Ошибка аутентификации 401 - ИСПРАВЛЕНО**
- Проблема: middleware искал токен в `req.cookies?.auth`
- Решение: теперь ищет в `req.cookies?.auth_token` (правильное имя cookie)
- Обновлены файлы:
  - `routes/paymentRoutes.js`
  - `routes/subscriptionRoutes.js` 
  - `routes/specialistsRoutes.js`
  - `middleware/magicAuthMiddleware.js`

### 2. **Ошибка сохранения специалиста - ИСПРАВЛЕНО**
- Проблема: если у пользователя нет подписки, система требовала email
- Решение: если подписки нет, считается SOLO план (email не требуется)
- Добавлены отладочные логи для диагностики

## 🚀 Как тестировать

### **Шаг 1: Проверка аутентификации**
```bash
# Проверьте что вы залогинены в системе
# Откройте консоль браузера и выполните:
localStorage.getItem('auth_token')
# Должен показать токен или null
```

### **Шаг 2: Тестирование платежей**
1. Зайдите в настройки (`/settings.html`)
2. Выберите тариф "1 специалист" (SOLO)
3. Нажмите "Оплатить"
4. Система автоматически создаст trial-подписку на 5 дней

**Ожидаемый результат:**
```
✅ Запись успешно создана
✅ Trial подписка активирована
✅ Специалист сохранен (только имя, email не требуется)
```

### **Шаг 3: Тестирование специалистов**
1. После оплаты тарифа, добавьте специалиста
2. Введите только имя (для SOLO плана email не нужен)
3. Нажмите "Сохранить"

**Ожидаемый результат:**
```
✅ Специалист создан успешно
✅ Отображается в списке специалистов
```

## 🔧 Тестовый режим

Система автоматически работает в тестовом режиме если:
```env
CLOUDPAYMENTS_PUBLIC_ID=your-cloudpayments-public-id
CLOUDPAYMENTS_API_SECRET=your-cloudpayments-api-secret
```

Если credentials не настроены, система:
- Создает trial-подписку на 5 дней для месячных планов
- Создает годовую подписку для годовых планов
- Не требует реальных платежей

## 🚨 Если все еще есть проблемы

### **Ошибка 401 Unauthorized:**
1. Проверьте `localStorage.getItem('auth_token')` в браузере
2. Перезайдите в систему
3. Очистите кэш и cookies

### **Ошибка сохранения специалиста:**
1. Проверьте логи сервера
2. Убедитесь что у вас есть бизнес (businessId)
3. Проверьте статус подписки

### **Проблемы с платежами:**
1. Проверьте что пользователь имеет `businessId`
2. Убедитесь что тариф существует в `PLANS`
3. Проверьте логи на наличие ошибок
